1. MVP 방향성

AkibaLens의 MVP는 사용자가 피규어 사진을 업로드하면 AI Vision Model이 상품 후보를 식별하고, 일본 쇼핑몰 가격 비교 결과를 제공하여 구매 판단을 돕는 end-to-end 흐름을 구현하는 것이다.

초기 버전의 목적은 당장 사업화가 아니라 실제 사용 가능한 제품을 빠르게 만들고, 사용자 니즈와 기술적 가능성을 검증하는 것이다. 따라서 자체 모델 학습, 복잡한 추천 시스템, 결제, 구매 대행 기능은 제외한다.

MVP에서 검증할 것
  • 피규어 사진만으로 유의미한 상품 후보를 제공할 수 있는가
  • AI가 생성한 검색 키워드가 실제 가격 검색에 도움이 되는가
  • 라이트 유저와 일본 여행객의 구매 판단을 줄여주는가
  • 2~4주 안에 배포 가능한 구조로 구현 가능한가
초기 제외 범위
  • 자체 CV 모델 학습, CLIP Fine-tuning, CNN 학습
  • 복잡한 추천 시스템
  • 결제 기능과 구매 대행 기능
  • 모든 쇼핑몰의 완전 자동 크롤링

4. 주차별 상세 계획

Week 1. 프로젝트 세팅과 이미지 업로드

구분 작업 내용 산출물
Frontend Next.js, TypeScript, Tailwind CSS 프로젝트 생성. 업로드 화면, 이미지 미리보기, 분석 버튼 UI 구현. 이미지 업로드 페이지
Backend FastAPI 프로젝트 생성. 이미지 업로드 endpoint, 파일 검증, 임시 분석 응답 endpoint 구현. 기본 API 서버
Infra .env.example, CORS 설정, 로컬 실행 스크립트 정리. 로컬 개발 환경

Week 2. Vision API 연동과 결과 UI

구분 작업 내용 산출물
AI OpenAI Vision 또는 Gemini Vision API 연동. 이미지에서 캐릭터명, 작품명, 제조사, 시리즈명, 추정 상품명 추출. 분석 결과 JSON
Prompt 피규어 식별 전용 프롬프트 설계. 불확실한 경우 후보와 신뢰도를 반환하도록 구조화. 프롬프트 템플릿
Frontend 분석 결과 카드, 후보 상품 리스트, 일본어/영어 검색 키워드 표시. 식별 결과 화면

Week 3. 가격 비교와 Marketplace Search 1차

구분 작업 내용 산출물
Price UI AmiAmi, Mandarake, Suruga-ya 기준 가격 비교 테이블 구현. 우선 mock 데이터로 UI와 데이터 구조 검증. 가격 비교 테이블
Search 식별 결과 기반 검색 URL 생성. 가능한 경우 검색 결과 수집 또는 외부 검색 링크 제공. 쇼핑몰 검색 연결
Database 분석 요청, 상품 후보, 가격 결과 저장을 위한 최소 PostgreSQL 스키마 설계. 초기 DB 스키마

Week 4. QA, 배포, 포트폴리오 정리

구분 작업 내용 산출물
QA 이미지 형식 오류, API 실패, 식별 실패, 가격 결과 없음 등 예외 처리. 안정화된 MVP
Deploy Frontend는 Vercel, Backend와 DB는 Railway 또는 Render에 배포. 접근 가능한 데모 URL
Portfolio README, 아키텍처, 기술 의사결정, 한계점, 향후 로드맵 정리. 취업용 포트폴리오 문서

backend/ FastAPI 기반 이미지 분석, Vision API 호출, 검색/가격 비교 API
docs/ 기획서, MVP 계획서, 아키텍처 문서, 발표 자료
scripts/ 개발 보조 스크립트, 데이터 수집 실험 코드
.env.example API key, DB URL 등 환경 변수 샘플

6. MVP 완료 기준

  1. 사용자가 웹에서 피규어 이미지를 업로드할 수 있다.
  2. Backend가 Vision API를 호출하여 구조화된 JSON 결과를 반환한다.
  3. Frontend가 캐릭터명, 작품명, 제조사, 시리즈명, 추정 상품명을 표시한다.
  4. AI 결과 기반 일본어/영어 검색 키워드를 제공한다.
  5. AmiAmi, Mandarake, Suruga-ya 기준 가격 비교 UI를 제공한다.
  6. 최소 3~5개의 실제 피규어 이미지로 데모 시나리오를 만들 수 있다.
  7. README에 문제 정의, 아키텍처, 실행 방법, 한계점, 향후 계획이 정리되어 있다.

7. 리스크와 대응

리스크 영향 대응 전략
Vision API가 정확한 상품명을 맞히지 못함 상품 식별 품질 저하 정답 하나가 아니라 후보 리스트, 신뢰도, 검색 키워드 중심으로 설계
쇼핑몰 크롤링이 막히거나 불안정함 가격 검색 자동화 지연 초기에는 검색 URL 생성과 mock 가격 비교 UI를 먼저 구현
구현 범위가 커짐 2~4주 내 MVP 실패 결제, 추천, 자체 모델 학습, 구매 대행은 명확히 제외
DB 설계가 과해짐 개발 속도 저하 초기에는 분석 결과와 가격 결과 저장에 필요한 최소 테이블만 설계
